(Prototype3.js)

함수는 객체이지만 일반 객체와는 다르게 함수 객체만의 표준 프로퍼티가 존재한다.
함수에는 내부의[[prototype]], prototype프로퍼티 2개가 존재한다.
=> prototype이라는 프로퍼티가 존재하는 것이다.

prototype 프로퍼티는 함수가 생성될 때, 만들어지며 두 개의 프로퍼티가 존재한다. constructor 프로퍼티와 내부 프로퍼티인 [[Prototype]]가 있으며, constructor 프로퍼티는 자신과 연결된 함수를 가르킨다.

1. 함수가 생성될 떄 JS엔진은 prototype 프로퍼티를 추가한다.
2. 함수의 prototype 프로퍼티는 함수의 프로토타입 객체를 가르킨다.
3. 함수의 프로토타입 객체에는 constructor 프로퍼티가 존재하는데, constructor 프로퍼티는 프로토타입 객체를 연결한 함수인 UserInfo를 가르킨다.
=> 정리하자면, 함수의 prototype 프로퍼티는 함수의 프로토타입 객체를 참조하고 함수의 프로토타입 객체의 constructor 프로퍼티는 연결된 함수를 참조한다.


내부 프로퍼티 [[prototype]]는 프로토타입 객체를 가르킨다.
내부 프로퍼티 [[Prototype]]은 객체, 배열에도 존재하며, 부모 역할을 하는 프로토타입 객체를 가르킨다.


prototype과 [[Prototype]]
내부 프로퍼티인 [[Prototype]]는 부모 역할을 하는 프로토타입 객체를 가리키며, 함수 객체에 존재하는 prototype 프로퍼티는 함수가 생성자로 사용될 때 생성된 객체의 부모 역할을 하는 프로토타입 객체를 가리킵니다.
=> prototype 프로퍼티는 생성자 함수로 사용했을 넘겨줄 프로토타입 객체이다.
