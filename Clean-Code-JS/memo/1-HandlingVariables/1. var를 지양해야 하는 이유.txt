1. var를 지양해야 하는 이유
=> var는 함수 스코프/ let, const는 블록 스코프를 가집니다.
=> let, const를 사용하면 안전한 코드를 작성할 수 있습니다.

1-1. var는 재선언이 가능하고 호출된 다음에 선언해도 undefined가 반환됩니다.
1-2. let, const는 한 번 선언하면 똑같은 변수명으로 선언할 수 없습니다.
1-3. const는 재 할당 할 수 없습니다.

** high
=> 원시값은 메모리에 값 자체를 저장합니다.
=> 참조값은 메모리 주소를 저장합니다. 그렇기에 배열이나 객체의 내부 값 변경을 하더라도
const는 에러를 내뿜지 않습니다.
=> 객체를 불변으로 만들고 싶다면 Object.freeze()를 사용해야 합니다. 그러나 중첩 객체나 중첩 배열의 내부의 값은
여전히 변경이 가능합니다.
=> immer 라이브러리를 사용하여 불변성을 만드는 것이 좋습니다.

=> const는 재할당이 불가능하다!
