1. 싱글톤 패턴
정의 : 특정 인스턴스가 하나만 존재하도록 보장하는 패턴을 의미한다.

1-1. constructor는 private이다. => new 키워드를 통해 인스턴스 생성 불가능
(최초 한번은 가능하게 만들 수 있다.)
1-2. static method를 통해 인스턴스에 접근한다. => 여러개의 인스턴스가 아닌 단 하나의 인스턴스만 사용 가능하도록 만든다.
1-3. 하나의 객체 인스턴스만 존재한다.
1-4. 그렇기에 객체의 주소값은 같다.
1-5. 일반 객체 리터럴도 싱글톤 패턴이다. => 객체의 주소값이 다르기 떄문이다.


2. 장점
2-1. 메모리 이점
2-2. 객체 인스턴스가 하나라서 속도적인 이점


3. 단점
3-1. JS에서는 객체를 바로 생성할 수 있기 떄문에 오버 코딩이다.
