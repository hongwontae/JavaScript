1. 배포 단계
웹 사이트에 배포할 수 있는 애플리케이션은 크게 두 종류로 나뉘어진다.
하나는 HTML+CSS+JS, 다른 하나는 HCJ+Node.js이다. (클라이언트만, 클라이언트+서버)
정적 배포와 비정적 배포라고 불린다.

코드 작성 => Test => Optimize => Build for Production => Deploy Output
=> 서버가 있다면 배포를 위한 성능개선이나 최적화 과정이 매우 축소된다. 서버에서 빨리 작동할 수 있기 떄문이다.

2. 예시 : 정적 호스트 배포(server code none)
webpack를 사용하면 최적화와 성능개선을 구현할 수 있다.

호스팅 => 어떤 서비스를 빌려서 사용한다는 말이다. 웹 호스팅(외부의 서버를 빌려서 기능을 사용한다.)
배포과정+도메인 연결을 도와주는 서비스 => FireBase를 사용했다.
=> 기존에는 자신만의 코드를 자신만의 브라우저에서 볼 수 있었는데 호스팅 서비스를 통해 다른 사용자들도 URL로 접근할 수 있게 된다.


3. 예시 : 동적 페이지 배포 (서버 측 코드 이용)
서버 호스팅을 도와주는 호스트가 필요하다. => Heroku
