1. JavaScript란?
런타임에 컴파일링되는 동적 프로그래밍 언어이다. 자바스크립트는 웹페이지를 동적으로 만들기 위해 탄생했다.

사용자가 웹페이지를 이용하려면 브라우저를 이용해야 한다. 그리고 웹페이지는 고유의 url주소를 가지고 있다.
사용자 => 브라우저 => 서버를 요청 Request => 서버 Response => 웹 페이지(url)
=> 이 과정을 동적으로 만들어주는 것이 JavaScript이다.

클라이언트 => 서버의 응답을 받는 곳/ 서버 => 클라이언트의 요청을 받는 곳

기존의 웹 페이지는 항상 새 페이지를 로드했다. => 새로고침했다.

해석형 언어이기에 전송 중에 컴파일이 된다. 컴퓨터가 읽을 수 있는 언어가 된다.
자바는 개발 중에  컴파일이 되고 자바스크립트는 실행되기 전에 컴파일이 된다.

호스팅 언어이기 때문에 여러 환경(여러 브라우저)에서 실행이 된다.

가장 중요한 특징은 동적으로 웹페이지를 만들 수 있다는 것이다.


2. Js의 컴파일이란?
개발자의 자바스크립트 코드가 작동하려면 브라우저 상에서 구현이 되는데 이 떄 자바스크립트 엔진을 거쳐야 한다.
자바스크립트 엔진은 브라우저에 내장되어 있다. 엔진의 역할은 코드의 구문 분석이다.
자바스크립트의 작동은 항상 운영 체제의 단일 스레드에서 진행된다.


3. 동적언어, 약한타입언어
동적 해석형 언어 => 미리 컴파일되지 않았음을 의미한다. 자바 같은 경우 문법 상의 오류가 발생하면 실행자체가 되지 않는다. 그러나 js의 경우 런타임에 컴파일이 되기에 실행자체는 된다.

약한타입언어 => js는 변수에 데이터 타입을 지정하지 않고 사용한다. 그렇기에 변수에 1,'asd' 이렇게 맘대로 바꿀 수 있다는 것이다.


4. JS의 언어의 다양성
JS는 프론트 엔드에서 주로 사용하며 백엔드에서도 사용이 가능하다.
웹 페이지를 동적으로 구현하는데 사용되기도 하고 Node.js같은 백엔드 서버에서도 사용된다.

호스팅 => 서버의 전체 혹은 일부를 이용할 수 있도록 임대해주는 서비스를 의미한다.
