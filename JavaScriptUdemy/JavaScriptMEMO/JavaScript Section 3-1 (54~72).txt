1. Color Theme, File Icon Theme
File => Preferences => Theme => ...


2. 단축키
File => Preferences => KeyBoard Shortcuts (모든 단축키를 보거나 검색하거나 설정하는 공간이다.)
ctrl+shift+space => 구문에 대한 간략한 해석을 보여준다.


3. Settings
File => Preferences => Setting

Setting검색창에 tab width을  검색하면 탭을 눌렀을 때 공백을 칸을 지정할 수 있다. 기본값은 2이다.
Setting 왼쪽 상단에는 User와 Workspace가 있다. User는 모든 환경의 설정이고 User는 현재 환경의 설정이다.
(User는 Workspace를 지정하지 않았다면 가져가는 설정이라고 생각하면 된다.)


4. Search
View => Search (검색의 대상은 파일이름 뿐만 아니라 파일 내용도 가능하다.)


5. Console.log()
개발자 도구 콘솔에 정보를 출력하는 빌트인(built-in) 명령어이다. 원하는 코드에 대한 로깅이 가능하다.


6. Error Console
google의 console창에는 만약 오류가 발생하면 오류 메시지를 보여준다. 오류의 내용과 오류가 어디서 발생했는지를 알려주고 Source창에서는 오류가 발생한 코드를 보여준다. 만약 함수가 정의되지 않은 함수를 작성했다면 코드상에서는 만들어낸 함수인지, 내장함수인지를 구분하지 못한다. 그러나 브라우저에서는 이를 실행시켜야 하기 때문에 함수를 찾고 없으면 오류를 발생시킨다. 이 오류가 콘솔창에 나타나는 것이다.

(html파일에 스크립트 태그 2개를 이용해서 연결시킬 수 있다. 이떄 밑에 있는 js파일은 위에 있는 js파일의 함수를 사용할 수 있다. 그러나 역은 성립하지 않는다.)


7. 중단점
개발자 도구 => source (여기서 중단점을 활용할 수 있다.)
source창에는 개발자가 작업한 코드가 있다. 또한 왼쪽에 코드 줄 번호가 존재한다. 줄 번호를 선택하면 파란색이 되는데 이 때 웹페이지에서 반응이 일어나면 파란색 줄 번호까지만 코드가 실행된다. => 오류 확인에 용이하다.

다음 함수 호출 => 밑에 줄로 내려가서 코드를 실행한다. 옆에는 각 값이 출력된다.
