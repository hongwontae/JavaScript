HTTP 요청을 보내면 보내는 정보는 HTTP Header와 HTTP Body를 거쳐서 전달된다.

HTTP 헤더는 Body의 요청과 응답에 대한 간략한 정보가 있다. => Blue Print
1. 일반헤더
=> 본문 컨텐츠와는 관련없고 요청/응답이 생성된 날짜나 HTTP 통신에 대한 일반적인 정보가 들어있다.
2. 요청/응답 헤더
HTTP 요청할 떄와 응답 받을 떄의 헤더의 정보는 다르다. 서버의 요청은 요청 헤더, 클라이언트가 받을 떄는 응답헤더이다. 요청 헤더는 요청한 URL, 메서드(GET..), 요청 생성에 사용된 브라우저 등이 들어간다. 응답 헤더는 브라우저에서 수신되며 컨텐츠에 사용된 인코딩, 응답할 떄 사용되는 소프트웨어의 정보등이 들어가있다.
3. 엔티티 헤더
실제 메시지나 응답 데이터의 대한 정보가 들어있다. (내용이 있는게 아니다.) 응답 데이터의 길이나 만료날짜, 인코딩등이 들어가 있다.

HTTP 본문
=> 실제 요청에 대한 내용이 들어가 있다. 요청한 리소스에 따라 JS,CSS,HTML,TXT 파일이나 json 등이 들어있다.
