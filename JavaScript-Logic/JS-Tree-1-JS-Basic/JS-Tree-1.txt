1. JavaScript란?
정의 : 프로토타입 기반 객체지향 언어이다.

특징
1-1. 런타임 : 코드가 실행되는 환경
1-2. V8 : Google에서 발명한 JS 엔진으로, Chrome 브라우저와 Node.js에서 사용한다. jit 컴파일러를 사용한다.


2. script 태그
정의 : HTML 안에 JS을 추가하기 위해 사용하는 태그

특징
2-1. <script> 태그 안에서 JS 코드를 작성할 수 있다.
2-2. <script> 태그의 src 속성을 통해 코드를 분리시킬 수 있다.
2-3. <script> 태그에 src가 존재하면 내부 코드는 무시된다.


3. 문(statement)
정의 : 어떤 작업을 수행하는 문법 구조와 명령어를 의미한다.

특징
3-1. 세미콜론과 줄바꿈으로 구분한다.
3-2. 줄 바꿈 => 세미콜론 자동 삽입
3-3. 문을 구분하지 않으면 원치않은 결과가 나올 수 있다.


4. 주석 => //


5. 엄격 모드
정의 : 2009년 이후 ES5에서 새로운 기능 추가되고 기존의 기능 중 일부가 변경된 모드를 활성화 시키는 명령어

특징
5-1. 'use strict'
5-2. 위의 명령어가 최상단에 존재하면 해당 스크립트는 엄격모드이다.
5-3. 함수 본문 최상단에 위치시킬 수 도 있다. => 이 떄는 해당 함수만 엄격모드이다.
5-4. 클래스와 모듈을 사용하면 자동으로 엄격모드가 적용된다.


6. 변수
정의 : 데이터를 저장하는 이름이 붙은 저장소이다. 키워드는 var, let, const가 존재한다.

특징
6-1. 문자, 숫자, $, _만으로 변수명을 작성해야 한다.
6-2. 첫 글자는 숫자면 안된다.
6-3. 대문자와 소문자를 구분한다.
6-4. 예약어는 변수명으로 사용할 수 없다.


7. 상수
정의 : 변화하지 않은 변수를 의미한다. 키워드는 const이다.

특징
7-1. 하드코딩한 값을 변수로 사용할 떄 대문자와 _를 조합해서 사용하는 경우가 많다.


8. 자료형
정의 : JS의 값은 항상 문자나 숫자 같은 특정한 자료형에 속한다. 8가지가 존재한다.

특징
8-1. 변수에 저장되는 값의 타입은 언제든지 바뀔 수 있다. 이를 동적 타입 언어라고 한다.
8-2. number, bigInt, string, boolean, null, undefinded, obj, symbol


9. typeof
정의 : 인수의 자료형을 반환하는 연산자이자 함수이다.

특징
9-1. 인수가 null이라면 obj 반환 => JS 설계상 오류
9-2. 인수가 함수라면 function을 반환 => obj가 반환되어야 하는데 JS 설계상 오류


10. alert, prompt, confirm
정의 : 사용자와 상호작용하기 위한 함수

특징
10-1. 사용자의 특정 행동을 하지 않으면 브라우저가 작동하지 않는다.
10-2. alert => 알림
10-3. prompt => 특정 값을 받아오기 위한 함수
10-4. confirm => ture, false 값을 받아오기 위한 함수
