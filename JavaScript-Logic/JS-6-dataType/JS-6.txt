1. 원시값 vs 참조값

원시값
콜스택에 값이 저장된다.
obj을 제외한 자료형이다.

참조값
힙 메모리에 값이 저장된다.
객체(배열, 함수)을 의미한다.    


2. 원시값+객체
원시타입에도 프로퍼티가 존재한다.
원시타입 메서드에 접근하면 래퍼 객체가 된다. => 메서드가 실행된다. => 래퍼 객체가 파괴되고 원시타입만 남는다.

*** 래퍼 객체를 new를 사용해서 사용하면 원시타입이 참조타입처럼 행동한다. => 혼동일으킨다.
*** 래퍼 객체를 new 없이 사용하면 단순히 형변환일 뿐이다.   


3. Number
isNaN(n) => 인수를 숫자로 변환한 다음 NaN인지 Boolean으로 반환
isFinite(n) => 인수를 숫자로 변환한 다음 NaN/Infinit가 아니라면 true를 반환

parseInt()/parseFloat() => 문자에서 숫자를 추출하는 메서드

Math
1. Math.random() => 0~1 사이의 난수
2. max
3. pow => 거듭제곱
4. floor, round => 소수점 처리
...

[] 표기법으로 문자 데이터 접근 가능
