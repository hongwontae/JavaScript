1. Scope란?
정의 : 식별자 접근 규칙에 따른 유효 범위를 의미한다.

특징
1-1. 식별자에 접근할 수 있는 범위(스코프)가 존재한다.
1-2. 범위는 중괄호(블록), 함수에 의해 나눠진다.
1-3. Block Scope, Function Scope라고 불린다.


2. 스코프의 주요 규칙
2-1. 안쪽 스코프에서 바깥쪽 스코프로 접근할 수 있지만 반대는 불가능하다.

2-2. 코프는 중첩이 가능하다.

2-3. 전역 스코프와 지역 스코프
가장 바깥쪽의 스코프를 전역 스코프(Global Scope)라고 부른다.
전역이 아닌 다른 스코프는 전부 지역 스코프(Local Scope)이다.

2-4. 지역 변수는 전역 변수보다 우선순위가 더 높다.
전역 스코프에서 선언한 변수는 전역 변수이다.
지역 스코프에서 선언한 변수는 지역 변수이다.
지역 변수는 전역 변수보다 더 높은 우선순위를 가진다.

2-5. 화살표 함수는 블록 스코프로 취급한다.


3. var, let, const Scope
var는 함수 스코프만 유효(블록 스코프 무시), let, const는 함수 스코프, 블록 스코프 둘 다 유효하다.


4. 스코프 체인
정의 : 스코프가 계층적으로 연결된 것을 의미한다.

특징
4-1. 변수를 참조할 때 JS 엔진은 스코프 체인을 통해 변수를 참조하는 코드의 스코프에서 시작하여 
상위 스코프 방향으로 이동하며 선언된 변수를 검색하는 것이다. (지역 스코프 => 전역 스코프) (프로토타입 체인과 비슷)

4-2. 스코프 체인은 실행 컨텍스트의 렉시컬 환경을 단방향으로 연결(Chaining)한 것이다.
