CallBack Hell

JS는 synchronous이다.
호이스팅된 이후 순서에 맞춰 하나씩 실행된다.
호이스팅 : var나 함수 선언문이 제일 위에서 등록된 것

콜백함수 
=> 매개변수로 함수를 받아서 기준에 충족하면 함수를 실행시키라는 의미이다.
=> 매개변수의 콜백함수의 제어권이 해당 함수에게 넘어갔다.


콜백은 무조건 비동기적 실행은 아니다. 비동기적 콜백과 동기적 콜백이 존재한다.
=> Browser에게 넘기는 브라우저api의 콜백함수는 비동기적 실행이다.
=> 일반적인 JS 자체 코드는 콜백함수라고 동기적으로 실행한다.

콜백지옥 => 콜백함수의 콜백함수의 콜백함수를 넣어서 비동기 처리 결과를 계속 넘기는 것이다.

** 인자의 함수를 건네준다. => 함수 호출의 인자의 함수가 실행되는 것이다.
** 해당 함수 본문과는 다르게 생각해야 한다.
